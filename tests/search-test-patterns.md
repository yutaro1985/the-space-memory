# ナレッジ検索 テストパターン

検索品質の回帰テスト・効果測定に使う標準クエリセット。
機能追加や検索アルゴリズム変更時にこのパターンで測定し、結果を比較する。

## 使い方

```bash
KS=./target/release/knowledge-search

# 全パターン一括実行
for q in "rust" "射撃" "LoRa トラッカー" "lindera" "先月の調査" "最近のメモ"; do
  echo "--- $q ---"
  $KS search -q "$q" -k 5
  echo ""
done
```

## テストパターン一覧

### 2. クエリ展開（Phase B: expand_query_entities）

### スコアリング品質（定量評価）

| 指標 | 測定方法 | 目標 |
|---|---|---|
| **Precision@5** | 上位5件中、関連ドキュメントの割合 | 80% 以上 |
| **MRR (Mean Reciprocal Rank)** | 最初の関連ドキュメントの順位の逆数の平均 | 0.7 以上 |
| **SCORE_THRESHOLD 超過率** | 検索結果数 / 候補数 | フィルタで適切にノイズが除外されているか |
| **レイテンシ** | クエリ応答時間（ms） | FTS5 のみ: < 50ms, 3ウェイ: < 200ms |
